Создай полный обучающий модуль по следующей теме для сотрудников розничного магазина.

Название темы: {{TOPIC_TITLE}}

Исходный контент:
{{TOPIC_CONTENT}}

Верни JSON-объект строго следующей схемы (без markdown-обёртки):

{
  "lecture_html": "<HTML-строка с контентом лекции>",
  "mindmap": {
    "nodes": [
      {"id": "1", "label": "Корневая концепция"},
      {"id": "2", "label": "Подтема 1"},
      ...
    ],
    "edges": [
      {"from": "1", "to": "2"},
      ...
    ]
  },
  "mindmap_md": "# Корень\n## Ветка 1\n### Лист 1.1\n## Ветка 2\n...",
  "quiz": [
    {
      "q": "Текст вопроса?",
      "options": ["Вариант А", "Вариант Б", "Вариант В", "Вариант Г"],
      "correct": 0,
      "explain": "Краткое объяснение правильного ответа."
    }
  ],
  "sim_brief": {
    "persona": "Описание персонажа-покупателя для ролевой игры",
    "need": "Что покупатель ищет или хочет узнать",
    "objections": ["Возражение 1", "Возражение 2", "Возражение 3"],
    "rubric": ["Критерий оценки 1", "Критерий оценки 2", "Критерий оценки 3", "Критерий оценки 4"]
  }
}

Требования к каждому полю:

lecture_html:
- ПЕРЕНЕСИ ВСЁ СОДЕРЖИМОЕ ИСТОЧНИКА — не сокращай, не пересказывай, не выбирай «главное»
- Никакого лимита слов: лекция должна содержать ВЕСЬ текст карточки товара
- Разрешённые теги: <h2>, <h3>, <p>, <ul>, <ol>, <li>, <strong>, <em>, <table>, <thead>, <tbody>, <tr>, <th>, <td>
- Нет обёртки <html>/<body>/<div>
- Каждый раздел источника (## заголовок) → <h2> в HTML
- Каждую markdown-таблицу (| col | col |) → HTML <table><thead>...<tbody>...</table> без исключения
- Каждый маркированный список → <ul><li>
- <strong> для ключевых терминов, важных правил, предупреждений (⚠)
- Сохраняй все эмодзи (✓, ✗, ⚠) в тексте

mindmap.nodes + mindmap.edges:
- 8–15 узлов, образующих дерево от корня (id "1")
- Корень = название темы, ветки = ключевые аспекты, листья = детали

mindmap_md:
- Markmap-совместимый markdown, воспроизводящий то же дерево
- Заголовки H1–H6 = уровень вложенности (#, ##, ###, и т.д.)

quiz:
- Ровно 5 вопросов
- Микс: фактические знания + применение на практике
- correct — индекс правильного варианта (0–3)
- explain — 1–2 предложения, объясняющих ПОЧЕМУ именно этот ответ

sim_brief:
- persona: реалистичный тип покупателя розничного магазина (пол, возраст, повод покупки)
- need: конкретный запрос покупателя, с которым он приходит в магазин
- objections: 2–4 реалистичных возражения / сомнения покупателя по теме
- rubric: 4–5 критериев оценки работы сотрудника в этом диалоге
